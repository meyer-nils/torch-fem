# Support

## Getting help

If you need help using torch-fem, please open a GitHub issue:

- Bug reports: use a clear title and include a minimal reproducible example.
- Usage questions: include your environment details and code snippet.
- Feature requests: describe the research use case and expected behavior.

Issue tracker:
https://github.com/meyer-nils/torch-fem/issues

## What to include in a support request

Please include:

- torch-fem version
- Python version
- OS and hardware (CPU/GPU)
- Installation method
- Full traceback or error output
- Minimal code example that reproduces the issue

## Response expectations

Support is provided on a best-effort basis.

- This is early-stage research software, so response times can vary.
- We generally prioritize reproducible bug reports and installation issues.
- Critical bug reports may be prioritized ahead of feature requests.

## Security issues

If you believe you found a security vulnerability, do not open a public issue.
Please report privately to:

`nils.meyer@uni-a.de`
